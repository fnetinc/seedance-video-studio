# Seedance Video Studio

Build long-form videos by chaining 15-second 16:9 clips. Each clip is generated by **Seedance 2.0** via [useapi.net](https://useapi.net) ‚Äî and the last frame of clip *N* is automatically extracted and used as the reference for clip *N+1*, so the sequence stays visually continuous. Designed to deploy on **Railway** from GitHub.

## Features
- **Login wall** ‚Äî username + password gate via `APP_USERNAME` / `APP_PASSWORD`
- **Chain mode** ‚Äî upload one starting frame + a list of prompts (one per line). The server runs them sequentially, threading each clip's last frame into the next.
- **Patient orchestration** ‚Äî Seedance generations can take 5‚Äì25 minutes; the server polls every 30s and waits up to 50 minutes per clip
- **Stop button** ‚Äî abort a running chain at any point; the in-flight clip finishes, then the chain halts
- **Live progress** ‚Äî overall progress bar, current clip status, polls counter, elapsed timers
- **Per-clip download + bulk `.zip`** ‚Äî every completed clip is cached on the server and available for individual download or one-shot bulk export
- **Explore mode** ‚Äî the API call sets `exploreMode: true` so generations don't burn credits (requires an unlimited Runway plan on the linked account)

## Stack
- Node.js + Express (single `server.js`)
- `@ffmpeg-installer/ffmpeg` + `fluent-ffmpeg` for last-frame extraction (no system install needed ‚Äî bundled binary)
- `multer` (in-memory upload), `archiver` (bulk zip)
- Vanilla HTML/CSS/JS frontend served from `/public`

## Prerequisites

useapi.net brokers Runway access through one or more **Runway accounts you link inside the useapi.net dashboard**. Before this app can generate anything:

1. Sign in at [useapi.net](https://useapi.net) and grab your bearer token.
2. Link at least one Runway account in their dashboard. Confirm via `GET /runwayml/accounts/`.
3. With **one** linked account, useapi.net auto-selects it. With **multiple**, it load-balances unless you pin one via the `RUNWAY_EMAIL` env var.

Once deployed, hit `GET /api/accounts` on your own server (after logging in) to confirm which Runway accounts useapi.net sees.

## Deploy to Railway

1. Push this repo to GitHub.
2. In Railway: **New Project ‚Üí Deploy from GitHub Repo** ‚Üí pick the repo.
3. Add environment variables:
   - `USEAPI_TOKEN` (required) ‚Äî your useapi.net bearer token
   - `APP_USERNAME` (required) ‚Äî frontend login
   - `APP_PASSWORD` (required) ‚Äî frontend login
   - `RUNWAY_EMAIL` (optional) ‚Äî pin a specific Runway account when multiple are linked
4. Railway auto-detects Node and runs `npm start`. Generate a public domain in service settings.

## API surface

All `/api/*` routes (except `/api/auth/login`) require **HTTP Basic** auth using `APP_USERNAME` / `APP_PASSWORD`.

| Method | Path                       | Purpose                                                              |
| ------ | -------------------------- | -------------------------------------------------------------------- |
| POST   | `/api/auth/login`          | `{username, password}` ‚Üí 200 if valid                                |
| POST   | `/api/chain/start`         | multipart: `image` + `prompts` (JSON array) ‚Üí returns `chainId`      |
| GET    | `/api/chain/status?chainId=‚Ä¶` | snapshot of chain state (current clip, status, polls, completed clips) |
| POST   | `/api/chain/stop`          | `{chainId}` ‚Üí marks chain to stop after current clip                 |
| GET    | `/api/chain/clip?chainId=‚Ä¶&index=‚Ä¶` | streams a single completed clip (mp4)                       |
| GET    | `/api/chain/zip?chainId=‚Ä¶` | streams a `.zip` of all completed clips                              |
| DELETE | `/api/chain?chainId=‚Ä¶`     | stop and forget the chain (frees server memory)                      |
| POST   | `/api/generate`            | one-shot: image + prompt ‚Üí `taskId`                                  |
| GET    | `/api/status?taskId=‚Ä¶`     | one-shot poll                                                        |
| GET    | `/api/download?taskId=‚Ä¶`   | one-shot download                                                    |
| GET    | `/api/accounts`            | lists linked Runway accounts useapi.net sees                         |
| GET    | `/healthz`                 | health check (no auth)                                               |

## How chaining works

```
[your starting image]
        ‚îÇ
        ‚ñº
   ‚îå‚îÄ Upload as asset
   ‚îÇ  Submit clip 1 with prompt[0]
   ‚îÇ  Poll Seedance every 30s until SUCCEEDED  (5‚Äì25 min typical)
   ‚îÇ  Download mp4
   ‚îÇ  ffmpeg: extract frame near end of clip
   ‚îÇ
   ‚îÇ ‚óÑ‚îÄ‚îÄ extracted frame becomes reference for clip 2
   ‚ñº
   [repeat for prompt[1], prompt[2], ‚Ä¶]
```

If the user clicks **Stop**, the orchestrator checks the flag at every awaitable step (poll loop, between phases) and exits cleanly. Already-completed clips remain available for download.

## Notes & limits
- All clip buffers are held in **server memory**. A Railway restart loses in-flight chains and completed clip buffers. For long-running production use, swap the in-memory `chains` Map for object storage (S3 / Railway volumes).
- Per-clip ceiling: 50 min. Per-chain max: 100 prompts. File-upload size: 25 MB.
- Frame extraction uses `ffmpeg -sseof -0.5` to grab a frame ~0.5s before the end. Reliable for mp4 produced by Runway/Seedance.
